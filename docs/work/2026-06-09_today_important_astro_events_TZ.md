# ТЗ: W-TODAY-IMPORTANT-ASTRO-EVENTS — блок «Сегодня важно»

Дата: 2026-06-09
Статус: ready for coder
Scope: backend-driven Today Important events + frontend rendering

## 1. Цель

Реализовать нормальный backend-driven блок `Сегодня важно`, который показывает пользователю короткий список реально значимых астрологических событий дня:

- Луна без курса с точным временем;
- новолуние / полнолуние;
- затмения;
- ретроградный Меркурий;
- станция Меркурия;
- четверти Луны;
- важные короткие аспекты быстрых планет.

Блок должен быть полезен “одним взглядом”, без перегруза и без долгих фоновых событий типа Плутона/Нептуна.

---

## 2. Главный принцип времени

Все события считаются и отображаются **в локальном времени пользователя**.

Источник timezone:

```text
profile.current_location.tz
```

Fallback:

```text
1. profile.current_location.tz
2. profile.birth_tz
3. Europe/Moscow
```

Для пользователя в Сочи показываем московское время. Для пользователя в Амстердаме — Europe/Amsterdam.

---

## 3. Что выводим

Whitelist событий для MVP:

```text
1. Луна без курса
2. Новолуние ±3 дня
3. Полнолуние ±3 дня
4. Солнечное затмение ±3 дня
5. Лунное затмение ±3 дня
6. Меркурий ретроградный
7. Станция Меркурия ±2 дня
8. Первая четверть Луны ±1 день
9. Последняя четверть Луны ±1 день
10. Солнце входит в новый знак
11. Важные короткие аспекты быстрых планет ±1 день
```

Если событий нет — блок `Сегодня важно` скрывается.

---

## 4. Что НЕ выводим

Не выводить в `Сегодня важно`:

```text
Плутон ретроградный
Нептун ретроградный
Уран ретроградный
Сатурн ретроградный
Юпитер ретроградный
долгие аспекты внешних планет
каждый лунный аспект подряд
Луна просто в знаке
минорные аспекты
астероиды
узлы, кроме затмений
```

## 5. Луна без курса

### 5.1. Правило расчёта

Луна без курса начинается после последнего мажорного аспекта Луны перед выходом из текущего знака и заканчивается входом Луны в следующий знак.

Мажорные аспекты:

```text
соединение
секстиль
квадрат
трин
оппозиция
```

Планеты для последнего аспекта:

```text
Солнце
Меркурий
Венера
Марс
Юпитер
Сатурн
Уран
Нептун
Плутон
```

### 5.2. Окно показа

Показывать, если интервал Луны без курса пересекает локальный день пользователя.

Минимальный порог:

```text
duration_inside_local_day >= 30 минут
```

Если интервал ночью и не влияет на день, можно понизить приоритет, но всё равно выводить, если он длительный или заходит в утро.

### 5.3. Формат

Показывать точное локальное время:

```text
Луна без курса: 03:38–11:33
```

Если началась вчера:

```text
Луна без курса: до 11:33
```

Если закончится завтра:

```text
Луна без курса: с 22:10 до завтра
```

### 5.4. Текст

Короткий пользовательский смысл:

```text
Лучше не запускать новое и не принимать решения на спешке. Подходит для завершения, рутины и отдыха.
```

---

## 6. Новолуние / полнолуние

Окно показа:

```text
exact day ±3 локальных дня
```

Приоритет:

```text
exact day > ±1 день > ±2/3 дня
```

Формат:

```text
Новолуние сегодня
Полнолуние завтра
Полнолуние было 2 дня назад
```

Текст для новолуния:

```text
День перезагрузки и нового цикла. Лучше не форсировать, а настроить направление.
```

Текст для полнолуния:

```text
Пик эмоций и ясности. Хорошо завершать, видеть результат, не перегревать конфликты.
```

---

## 7. Затмения

Окно показа:

```text
exact day ±3 локальных дня
```

Типы:

```text
solar_eclipse
lunar_eclipse
```

Формат:

```text
Солнечное затмение сегодня
Лунное затмение через 2 дня
```

Текст:

```text
Сильная точка разворота. Лучше не принимать резких решений на эмоциях.
```

Затмение всегда выше по приоритету, чем обычное новолуние/полнолуние.

---

## 8. Меркурий ретроградный

Показывать только Меркурий. Другие ретроградности в Important не выводим.

События:

```text
mercury_retrograde_active
mercury_station_retrograde
mercury_station_direct
```

### 8.1. Retro active

Показывать весь период ретроградности:

```text
Меркурий ретроградный
```

Текст:

```text
Проверяй договорённости, документы, технику и сообщения. Лучше перепроверить детали.
```

### 8.2. Station

Окно станции:

```text
±2 дня
```

Станция важнее обычного retro active.

Формат:

```text
Меркурий разворачивается
```

Текст:

```text
Самая нестабильная точка периода: больше путаницы, задержек и пересмотров.
```

---

## 9. Четверти Луны

Показывать:

```text
first_quarter ±1 день
last_quarter ±1 день
```

Текст для первой четверти:

```text
День действия и проверки намерений. Может быть напряжение между желанием и реальностью.
```

Текст для последней четверти:

```text
День корректировки и освобождения от лишнего. Лучше завершать, чем начинать.
```

Приоритет ниже новолуния/полнолуния.

---

## 10. Солнце входит в знак

Показывать только день ингресса Солнца:

```text
sun_enters_sign exact local day only
```

Формат:

```text
Солнце входит в Рак
```

Текст:

```text
Меняется общий фокус месяца. День подходит, чтобы заметить новую тему и перестроить планы.
```

Особо отметить кардинальные точки:

```text
Овен
Рак
Весы
Козерог
```

Но без отдельного большого блока.

## 11. Важные короткие аспекты быстрых планет

### 11.1. Планеты

Быстрые планеты-инициаторы:

```text
Солнце
Меркурий
Венера
Марс
```

Планеты-адресаты:

```text
Марс
Юпитер
Сатурн
Уран
Нептун
Плутон
```

### 11.2. Окно показа

```text
exact aspect ±1 локальный день
```

### 11.3. Напряжённые аспекты

Выводить:

```text
соединение
квадрат
оппозиция
```

Только если аспект действительно заметный для бытового дня.

Примеры:

```text
Меркурий квадрат Нептун
Венера квадрат Сатурн
Марс квадрат Сатурн
Солнце квадрат Марс
Меркурий оппозиция Уран
Венера оппозиция Плутон
Марс квадрат Плутон
```

Тип:

```text
caution
```

Текст:

```text
Осторожнее с резкими решениями, конфликтами и обещаниями. Лучше перепроверить детали.
```

### 11.4. Позитивные аспекты

Выводить меньше, только очевидные “окна возможностей”:

```text
Солнце тригон/секстиль Юпитер
Меркурий тригон/секстиль Юпитер
Венера тригон/секстиль Юпитер
Марс тригон/секстиль Юпитер
Венера тригон/секстиль Солнце
```

Тип:

```text
supportive
```

Текст:

```text
Хорошее окно для переговоров, поддержки, знакомств, денег или продвижения дел.
```

### 11.5. Не выводить

Не выводить все подряд “приятные” аспекты, чтобы Important не стал шумным.

---

## 12. Типы событий

Каждое событие должно иметь тип:

```text
supportive
caution
neutral_shift
```

Примеры:

```text
void_moon => caution
new_moon => neutral_shift
full_moon => neutral_shift / caution
eclipse => caution
mercury_retrograde => caution
mercury_station => caution
sun_ingress => neutral_shift
venus_jupiter_trine => supportive
mars_saturn_square => caution
```

---

## 13. Приоритет сортировки

Показывать максимум:

```text
max 3 события
```

Сортировка:

```text
1. Затмение
2. Новолуние / полнолуние
3. Луна без курса
4. Станция Меркурия
5. Меркурий ретроградный
6. Важные напряжённые аспекты быстрых планет
7. Важные supportive аспекты быстрых планет
8. Солнце входит в знак
9. Первая / последняя четверть Луны
```

Если два события одного типа — ближе к exact time выше.

## 14. Backend contract

Добавить/уточнить структуру Important event в TodayPayload.

Пример TypeScript-формы контракта:

```ts
type TodayImportantEvent = {
  id: string
  kind:
    | "void_moon"
    | "new_moon"
    | "full_moon"
    | "solar_eclipse"
    | "lunar_eclipse"
    | "mercury_retrograde"
    | "mercury_station"
    | "moon_quarter"
    | "sun_ingress"
    | "fast_planet_aspect"

  tone: "supportive" | "caution" | "neutral_shift"

  title: string
  summary: string

  startsAt?: string
  endsAt?: string
  exactAt?: string

  localTimeLabel?: string
  timezone: string

  priority: number
}
```

Для Луны без курса обязательно:

```text
startsAt
endsAt
localTimeLabel
timezone
```

Пример payload:

```json
{
  "kind": "void_moon",
  "tone": "caution",
  "title": "Луна без курса",
  "summary": "Лучше не запускать новое. Подходит для завершения и рутины.",
  "startsAt": "2026-06-09T03:38:00+03:00",
  "endsAt": "2026-06-09T11:33:00+03:00",
  "localTimeLabel": "03:38–11:33",
  "timezone": "Europe/Moscow",
  "priority": 80
}
```

## 16. Frontend

Фронт должен только рендерить события из backend payload.

Не считать события на фронте.

UI:

```text
Сегодня важно
[icon] Луна без курса · 03:38–11:33
Лучше не запускать новое. Подходит для завершения и рутины.
```

Если `importantEvents.length === 0`, блок скрыть.

Если событий больше 3, frontend всё равно может отрендерить только первые 3, но лучше обрезать на backend.

## 17. Тесты

Добавить backend tests.

### 17.1. Void moon

```text
Given local day has void moon 03:38–11:33 Europe/Moscow
Then importantEvents includes void_moon
And localTimeLabel == "03:38–11:33"
And timezone == "Europe/Moscow"
```

### 17.2. Void moon crossing midnight

```text
Given void starts previous day and ends today 09:00
Then label == "до 09:00"
```

### 17.3. New moon / full moon

```text
Given date within ±3 days of full moon
Then full_moon is included
```

### 17.4. Eclipse priority

```text
Given eclipse and full moon same period
Then eclipse priority is higher
```

### 17.5. Mercury

```text
Given Mercury retrograde active
Then mercury_retrograde is included
```

```text
Given Mercury station within ±2 days
Then mercury_station is included and ranks above mercury_retrograde
```

### 17.6. Max 3

```text
Given 5 possible important events
Then only top 3 are returned
```

### 17.7. No events

```text
Given no whitelisted events
Then importantEvents is empty
And frontend hides block
```

## 18. Acceptance criteria

Пакет принимается, если:

```text
[ ] Луна без курса считается и выводится с точным локальным временем.
[ ] Timezone берётся из profile.current_location.tz.
[ ] Есть fallback на birth_tz / Europe/Moscow.
[ ] Новолуние и полнолуние выводятся в окне ±3 дня.
[ ] Затмения выводятся в окне ±3 дня и имеют высокий приоритет.
[ ] Показывается только ретроградный Меркурий, остальные ретроградности игнорируются.
[ ] Станция Меркурия выводится в окне ±2 дня.
[ ] Четверти Луны выводятся в окне ±1 день.
[ ] Солнце входит в знак выводится только в exact day.
[ ] Важные аспекты быстрых планет выводятся только по whitelist.
[ ] Максимум 3 события.
[ ] Если событий нет, блок скрывается.
[ ] Frontend ничего не рассчитывает, только рендерит backend payload.
[ ] Tests added for void moon, moon phases, Mercury, priority, max 3.
```

## 19. Не делать

Не добавлять:

```text
Плутон/Нептун/Уран/Сатурн/Юпитер ретроградный
долгие аспекты внешних планет
минорные аспекты
астероиды
лунные аспекты каждый час
натальную персонализацию в этом блоке
```

Персонализация дня идёт в основном Today-разборе, а `Сегодня важно` — это короткий общий астрологический alert.
