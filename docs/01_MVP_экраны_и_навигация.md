# 01. MVP: экраны и навигация

## Роль документа

Фиксирует состав MVP, нижнюю навигацию, маршруты и переходы между экранами.

## Состав MVP

1. Onboarding
2. Today / экран дня
3. Calendar / календарь
4. Readings / разборы
5. Profile / профиль
6. Soft lock / Paywall states

## Нижняя навигация

- **Сегодня** — главный экран продукта.
- **Календарь** — навигация по дням месяца, не месячный прогноз.
- **Разборы** — витрина глубоких форматов.
- **Профиль** — данные, доступ, рефералка, хорарные кредиты, настройки.

## Главная модель

**Today = экран выбранного дня.**

Экран Today — универсальный экран любой выбранной даты.

Он открывается для:

- сегодняшнего дня;
- вчерашнего дня;
- будущего дня;
- даты из календаря;
- даты из 7-дневной ленты;
- даты по стрелкам;
- даты по свайпу;
- deeplink / route.

## Route model

- `/` — onboarding или редирект на сегодняшний день;
- `/day/:date` — экран выбранного дня;
- `/calendar` — календарь;
- `/readings` — разборы;
- `/profile` — профиль.

Дата должна жить в URL. Это нужно для тестов, deeplink, back navigation, кэша backend и понятной связи UI/API.

Route отвечает на вопрос **«Какой день открыт?»**. Пользователь определяется Telegram auth/session, не route.

## Переходы между днями

1. Стрелка влево / вправо.
2. Swipe влево / вправо.
3. 7-дневная лента.
4. Календарь.

При смене дня:

- обновляется route;
- обновляется дата;
- обновляются флаги, текст, why-блок, активная плашка;
- экран скроллится наверх.

## 7-дневная лента

Это навигация, не недельный прогноз.

В каждой плашке:

- день недели;
- число;
- статус дня.

Статусы MVP:

- **поддерживающий**
- **ровный**
- **напряжённый**

Не используем много тематических тегов вроде «финансы», «контакты», «сборка», потому что они смешивают состояние, тему и рекомендацию. Темы дня остаются внутри Today.

## Календарь

Календарь — навигация по дням месяца, не месячный прогноз.

Диапазон MVP:

- прошлый месяц;
- текущий месяц;
- следующий месяц.

Тап по дню открывает `/day/:date`.

## Разборы

Раздел **«Разборы»** — витрина глубины продукта.

Доступные / потенциально доступные карточки:

- Натальная карта;
- Хорар.

Карточки **«в разработке»**:

- Месяц;
- Год;
- Синастрия;
- Тематические разборы;
- История / архив.

## Профиль

Профиль — центр управления:

- личными данными;
- локациями;
- доступом;
- подпиской;
- рефералкой;
- хорарными кредитами.

В профиле можно менять:

- дату рождения;
- время рождения;
- место рождения;
- текущий город проживания;
- город ближайшего дня рождения, если отличается.

## Onboarding

Собираем:

- дата рождения;
- время рождения;
- место рождения;
- текущий город проживания;
- город ближайшего дня рождения, если отличается.

После завершения пользователь попадает на `/day/:today`.

## Access states

Если доступ активен — полный контент доступных дней.

Если доступ закончился — навигация остаётся доступной, но полный контент закрывается через soft lock / paywall.

## Locked day behavior

Если пользователь открывает закрытый день:

- открывается тот же `/day/:date`;
- показывается preview;
- основной текст и «Почему так у меня» закрыты;
- показывается soft lock / paywall.

Locked day должен продавать ценность, а не быть мёртвой датой.

## Soft lock принцип

Не просто «Доступ закрыт», а:

> Этот день уже рассчитан для тебя. Полный разбор и объяснение доступны по подписке или приглашению друга.

CTA:

- оформить подписку;
- пригласить друга.
