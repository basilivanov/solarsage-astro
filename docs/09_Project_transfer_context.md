# 09. Project transfer context

## Назначение

Этот файл — общий контекст для переноса проекта в другой чат или передачи новому агенту.

Он кратко собирает все ключевые решения, текущий статус и следующие шаги.

---

# Что за проект

Telegram Mini App с персональным астрологическим ассистентом.

Это не общий гороскоп. Продукт должен давать человеку ощущение:

**«Это рассчитано именно про меня и объясняет мой день, состояние, фокус, трудности и опоры».**

Главный экран MVP — **Today screen**.

---

# Главный принцип продукта

Продукт отвечает не на вопрос:

**«Что происходит на небе?»**

А на вопрос:

**«Как это проявляется у меня?»**

Любой астрологический фактор должен переводиться в:

- состояние;
- дела;
- разговоры;
- фокус;
- трудности;
- поддержку;
- практический способ прожить день.

---

# MVP

В MVP входят:

- Onboarding;
- Today screen;
- Calendar;
- Readings;
- Profile;
- soft lock / paywall.

Нижняя навигация по документам:

- Сегодня;
- Календарь;
- Разборы;
- Профиль.

Во фронтовом макете появилась вкладка **Спросить** — это новое решение, его нужно отдельно принять или убрать.

---

# Today screen

Структура:

1. Дата + стрелки;
2. верхние флаги «Сегодня важно учесть»;
3. основной текст дня;
4. раскрытие «Почему так у меня»;
5. 7-дневная лента;
6. нижняя навигация.

Экран длинный, со скроллом. Главная ценность — текст на 3–5 минут чтения.

---

# Верхние флаги

2–4 строки. Лучше 3.

Каждая строка:

- иконка / маркер;
- заголовок;
- короткая расшифровка;
- стрелка справа;
- по тапу раскрывается inline hint.

Флаги не открывают отдельный экран.

Примеры:

- Луна без курса до 17:40;
- Меркурий ретрограден · ещё 9 дней;
- Активен 10 дом.

---

# Почему так у меня

Это inline-раскрытие внутри Today, не отдельный экран.

Структура:

1. Главная тема дня;
2. Быстрый слой дня;
3. Почему это задевает именно тебя;
4. Фон периода;
5. Что усиливает этот день;
6. Что смягчает этот день;
7. Через какие сферы это проявляется;
8. Астрологический смысл дня;
9. Что это значит practically.

Стиль: много астрологического мяса, дома, соляр, профекции, дирекции, прогрессии, транзиты, усилители, смягчители.

Запрещён стиль «если». Пишем итог расчёта как утверждение.

---

# 7-дневная лента

Для MVP используем 3 статуса:

- поддерживающий;
- ровный;
- напряжённый.

Не используем много тематических тегов вроде «финансы», «контакты», «сборка».

Темы дня остаются внутри Today.

---

# Календарь

Календарь — это навигация по дням, не месячный прогноз.

Диапазон MVP:

- прошлый месяц;
- текущий месяц;
- следующий месяц.

Тап по дню открывает `/day/:date`.

Дни вне разрешённого диапазона должны быть disabled.

---

# Доступ и монетизация

Постоянного free-tier нет.

Основной бесплатный вход:

**пригласи друга — вы оба получите 14 дней доступа.**

Правило списания:

1. сначала тратятся подарочные / реферальные дни;
2. потом подписка;
3. если подписки нет — доступ закрывается.

Без доступа интерфейс остаётся видимым, но полный контент закрыт через soft lock / paywall.

Locked day должен открываться как preview + paywall, а не быть мёртвой датой.

---

# Хорар

Хорар входит в продуктовую карту и раздел «Разборы».

Правила:

- 1 хорарный вопрос в неделю активного доступа;
- бонусные хорары копятся;
- позже хорары можно продавать отдельно.

В MVP хорар может быть доступен или помечен как «в разработке».

---

# SolarSage и расчёт

Подход:

**считаем максимально много, но показываем только стабильный продуктовый слой.**

Слои:

- SolarSage raw calculations;
- NatalDeepSnapshot;
- PeriodSnapshot;
- DailySnapshot;
- Normalized AstroSignal[];
- SemanticLayer;
- LLM interpretation;
- TodayPayload.

Формула:

```text
Day v1 = DailySky + PersonalActivations + PeriodBackground + NatalSensitivity + QualityModifiers
```

---

# Что учитывается на Today

Today использует:

- Луну;
- Луну без курса;
- фазы Луны;
- смену знака Луны;
- затмения;
- retro / station;
- точные транзиты;
- транзиты к наталу;
- транзиты к углам;
- соляр;
- лунар, если включаем;
- профекции;
- дирекции;
- прогрессии;
- solar arc;
- firdaria;
- long-term транзиты;
- lots / midpoints / fixed stars, если они точно активированы.

Но не всё показывается напрямую. Быстрый слой объясняет «почему сегодня», периодные техники объясняют «почему это сейчас важно именно для меня».

---

# Скоринг

Нельзя считать только:

```text
netScore = supportScore - frictionScore
```

Нужно считать:

- supportScore;
- frictionScore;
- intensityScore;
- netScore;
- shadowRisk;
- totalSalience;
- phase: building_up / exact / releasing.

Контрастный день не должен превращаться в «ровный» из-за взаимного обнуления поддержки и трения.

Для тихих дней нужен fallback на daily sky, а не натягивание слабого аспекта на главный сюжет.

---

# LLM

LLM не получает raw SolarSage.

Она получает curated SemanticLayer:

- main theme;
- top flags;
- daily triggers;
- personal activations;
- period background;
- amplifiers;
- softeners;
- manifestation zones;
- practical direction.

LLM не должна упоминать:

- salience;
- tensionScore;
- supportScore;
- raw JSON;
- backend evidence;
- internal ranking.

---

# API contracts

Главный endpoint:

```http
GET /api/day/:date
```

Фронт должен работать с `TodayPayload`, а не с raw SolarSage.

Стабилизируем форму выдачи, а не расчётную формулу.

---

# Натальная карта

Натальный разбор строится как versioned document:

```text
schemaVersion: "natal/v1"
sections[].blocks[]
block.type → component
```

Structured data отдельно от narrative:

- meta;
- highlights;
- spheres;
- planets.

Навигация:

- `/readings/natal` — overview / TOC;
- `/readings/natal/[section]` — отдельная глава.

Unknown block type не должен ломать экран.

# Главное правило для нового чата / агента

Не придумывать продукт заново.

Сохранять утверждённые решения:

- Today = экран выбранного дня;
- `/day/:date`;
- 4 вкладки в документах;
- long text first;
- флаги с inline hint;
- «Почему так у меня» inline;
- фронт не знает астрологическую кухню;
- backend отдаёт стабильный payload;
- LLM пишет только по SemanticLayer;
- расчёты можно менять, если контракты стабильны.
