# W-4.3 — SemanticLayer assembly + cache

status: completed
phase: PHASE-4-SCORING
wave: W-4.3
modules: M-SEMANTIC-SERVICE

---

## Decision

Create semantic layer that assembles scored signals into structured semantic blocks for LLM consumption. Cache semantic layer output to avoid recomputing for repeated requests.

**Why:** LLM needs structured, semantically rich input. Semantic layer bridges scoring output and prompt context. Caching reduces latency for calendar views and repeated day requests.

---

## Acceptance Criteria

- [x] `SemanticService` assembles semantic blocks from scored signals
- [x] Semantic blocks include day status, sphere scores, top signals
- [x] Semantic layer output cached by (user, date)
- [x] Cache invalidation on profile updates
