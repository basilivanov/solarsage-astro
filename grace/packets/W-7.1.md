## Why this wave exists

Natal reading is a core feature of SolarSage Astro. Users need a structured, 
detailed interpretation of their natal chart. This wave establishes the data 
structure for natal readings: sections (Sun, Moon, Ascendant, etc.) containing 
blocks of different types (paragraph, bullets, highlights, quote).

The existing schema in `apps/api/app/schemas/natal.py` already defines a rich 
block system with discriminated unions. This wave implements the service layer 
that generates natal reading payloads using this schema.

## Decision

Structure natal reading as sections (Sun, Moon, Ascendant) containing blocks 
of text. Use existing NatalPayload schema with:
- NatalMeta: schema version, title, subtitle, person info
- NatalSection: id, title, icon_name, blocks[]
- NatalBlock: discriminated union (paragraph | bullets | highlights | quote)

Service returns hardcoded MVP content. Real LLM integration is a future wave.
