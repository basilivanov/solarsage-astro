## Why this wave exists

With the natal reading data structure in place (W-7.1), users need API 
endpoints to retrieve natal readings. This wave exposes two endpoints:
- GET /api/natal/overview — returns meta + section list (for navigation)
- GET /api/natal/section/{section_id} — returns full section with blocks

This allows the frontend to:
1. Fetch overview to show section navigation
2. Fetch individual sections on demand (lazy loading)

## Decision

Two endpoints:
- GET /api/natal/overview — returns meta + section list (id, title, iconName)
- GET /api/natal/section/{section_id} — returns section with blocks

Both require authentication (require_session dependency).
Section endpoint returns 404 for non-existent section_id.
