# W-4.1 — Normalization to AstroSignal[]

status: completed
phase: PHASE-4-SCORING
wave: W-4.1
modules: M-NORMALIZATION-SERVICE

---

## Decision

Create normalization layer that transforms raw sidecar output (natal charts, transits) into unified `AstroSignal[]` format. Each signal represents a single astrological event (aspect, transit, ingress) with standardized fields (type, planet, aspect_type, strength).

**Why:** Sidecar returns heterogeneous data structures. Normalization creates a uniform interface for the scoring layer, decoupling calculation format from scoring logic.

---

## Acceptance Criteria

- [x] `AstroSignal` schema defined in `app/schemas/normalization.py`
- [x] Normalization service transforms sidecar output to `AstroSignal[]`
- [x] All signal types supported (aspects, transits)
- [x] Strength calculation normalized (0.0 to 1.0 range)
