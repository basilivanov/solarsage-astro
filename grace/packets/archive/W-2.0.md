## Why this wave exists

Backend already conforms to canon §7 / §11:

- `apps/api/app/api/health.py` carries `AI_HEADER`, `START_MODULE_CONTRACT`,
  `START_MODULE_MAP`, `START_FUNCTION_CONTRACT`, `START_BLOCK: HEALTH_PAYLOAD`.
- `apps/api/app/schemas/*.py` is the single source of truth; drift gate
  (W-1.1B) keeps `packages/contracts/_generated.ts` byte-identical.
- `bootstrap.sh` runs ruff, mypy, alembic, pytest, drift-grep and
  contracts idempotency on every change.

Frontend has none of this. Today `app/page.tsx` is empty; the moment
W-1.2 (`/day/today` pilot) starts producing `app/(today)/**`,
`components/today/**`, `lib/api/**` and route handlers, agents will
begin writing TSX/TS files **without** module contracts, **without**
semantic block markers, **without** an enforced rule that response
shapes come from `_generated.ts`, and **without** a structured-log
envelope on server actions and route handlers.

Result: reviewer-agent failure packets degenerate from
`module → block → invariant violated` to "something somewhere in this
500-line page", and the contract drift gate (W-1.1B) gets silently
bypassed by locally-redeclared `interface TodayPayload { ... }`.

This wave installs the missing rails **before** the first feature TSX
is written. No UI behaviour, no route, no page is added in this wave.

## Decision required from controller (record here, then proceed)

Pick exactly one option and write it under `## Decision` below.

- **A. Lint-only enforcement.** Add ESLint rules that require AI_HEADER
  + paired MODULE_CONTRACT/MODULE_MAP markers in `app/**`, `components/**`,
  `lib/api/**`, `app/api/**/route.ts`, plus a no-redeclare rule for
  contract types. Lint runs in `bootstrap.sh` as Gate 7. Pro: stays
  inside the JS/TS toolchain. Con: lint failures are easy to silence
  with eslint-disable; markers themselves are not parsed for content.

- **B. Bash grep gates only (mirror of backend §13.E).** Add Gate 7 to
  `bootstrap.sh` that greps required headers and balanced
  START/END markers in declared frontend roots, and a Gate 8 that
  greps for forbidden patterns (`interface TodayPayload`, etc.).
  Symmetric to how backend invariants are checked today. Con: no
  TS-level guarantee on import sources.

- **C. Both — lint for TS-aware rules (forbidden imports, redeclared
  shapes), bash grep for canon-marker presence and balance.**
  Pro: lint catches what greps cannot (cross-file type identity),
  greps catch what lint cannot (file-level header presence in a way
  that survives `eslint-disable-next-line`). Con: two systems to
  maintain.

The remainder of this packet is conditional on the decision. Allowed
Write Scope below covers the union; worker uses only the subset
matching the chosen option.

## Decision

**Variant: C (lint + bash grep gate).**

Rationale: AI-driven development without human diff review requires
defense in depth. ESLint covers AST-level contract drift
(import sources, locally redeclared payload types, cross-file type
identity) which grep cannot see. The bash gate in `bootstrap.sh`
covers comment-level GRACE markers (`AI_HEADER:`, paired
`START_MODULE_CONTRACT/END_MODULE_CONTRACT`, paired
`START_BLOCK: NAME / END_BLOCK: NAME`) which ESLint does not
naturally parse. Both gates must be green before a worker packet is
accepted. Enforced paths are an explicit whitelist (`grace/frontend.paths`),
not an exclusion list — adding a new directory is an intentional
controller action, not a default.
