# Controller Packet — W-2.8: JS test gate (vitest for `lib/grace/**`)

**STATUS: DEPRECATED — Merged into W-2.0**

This wave originally planned to add vitest gate as Gate 11 after W-2.7.
However, this created a gap where reducers written in W-2.4 and W-2.7 
remained untested until W-2.8 shipped.

**Resolution:** Vitest gate (Gate 11) was moved to W-2.0 (Frontend GRACE 
Conformance). This allows reducers to be tested immediately when written, 
eliminating the test coverage gap.

**For historical context, see below. For current implementation, see W-2.0.**

---

## Original Why this wave exists

By the end of W-2.7 the tree contains at least two reducer test files
under `lib/grace/**` (chat, onboarding, both ported from
`legacy/frontend/lib/reducers/reducers.test.ts`) and one pure-function
test file (port of `legacy/frontend/lib/pure-functions.test.ts`,
likely `lib/grace/pure-functions.test.ts` or relocated to per-domain
files). These tests exist but no runner picks them up. From a GRACE
perspective, an untested reducer carrying `MODULE_CONTRACT.invariants:
[...]` is a lie of omission; W-2.0's marker gate cannot prove that
invariants hold.

This wave lights up the runner so the invariants are checked on every
commit. Backend already has `pytest` as Gate 5; this wave installs the
symmetric Gate 11 for JS tests and stops there.

The wave does **not** add product tests, integration tests, or e2e.
Only the runner, its config, and a single appended bootstrap gate.
W-2.4 / W-2.5 / W-2.7 packets all noted that their `*.test.ts` files
were inert until W-2.8 — those files become live the moment this
wave's gate flips green.

## Decision required from controller (record here, then proceed)

Pick exactly one option and write it under `## Decision` below.

- **A. Vitest, root config, jsdom for component tests, node for pure
  modules; runs as Gate 11 in `bootstrap.sh`.** Pro: matches legacy
  setup (`legacy/frontend/vitest.config.ts`); shortest path. Con:
  jsdom adds a dependency footprint.

- **B. Vitest, node-only environment.** Forbid component tests in
  `lib/grace/**`; only pure-function and reducer tests are run.
  Component tests are pushed to e2e (Playwright, separate wave).
  Pro: smaller dep footprint; cleaner separation (unit ↔ e2e). Con:
  forces a future wave for component coverage; `lib/grace/**` already
  has at least one widget-level test in legacy that would have to
  move to e2e.

- **C. Bun test runner.** Replaces vitest entirely. Pro: zero
  config; ships with bun. Con: depends on the project package
  manager being bun; existing repo uses pnpm; this would be a tooling
  change touching CI.

The remainder of this packet is conditional on the decision. Default
expectation is **Option A** (vitest + jsdom; symmetric to legacy).

## Allowed Write Scope (assumes Option A; deltas for B/C noted)

### Test runner config

- `vitest.config.ts` (root) — module M-WEB-TEST-GATE.
  - test glob: `lib/grace/**/*.test.{ts,tsx}` and
    `components/grace/**/*.test.{ts,tsx}` (Option A).
  - environment: per-file or two projects (`unit` = node,
    `dom` = jsdom). Worker chooses one shape and records under
    `## Decision`.
  - reporter: default; CI flag from bootstrap.
  - exclude: `legacy/**`, `node_modules/**`, `.next/**`,
    `apps/api/**`.
- `vitest.setup.ts` — only if jsdom is enabled. Provides the minimal
  test-library setup (jest-dom matchers, etc.).
- `package.json` — devDependencies: `vitest`, `@vitest/ui`
  (optional, dev-only), `jsdom`, `@testing-library/react`,
  `@testing-library/jest-dom` (Option A). Plus a `test:grace` npm
  script. The script must be **idempotent**: `pnpm test:grace --run`
  exits with the same code on consecutive invocations on a clean
  tree.

### Bootstrap gate

- `bootstrap.sh` — append `=== Gate 11: vitest (lib/grace + components/grace) ===`
  after Gate 10. The block runs `pnpm exec vitest run --reporter=
  default` (or equivalent for chosen runner) and fails on non-zero
  exit. Append-only; no existing gate is touched.

### GRACE markers in test files

- Each ported `*.test.ts` carries the standard marker set
  (AI_HEADER, MODULE_CONTRACT, MODULE_MAP, FUNCTION_CONTRACT for
  every `it.*`/`test.*` block-helper if extracted, plus
  START/END_BLOCK around each `describe`). The marker gate already
  covers them via existing whitelist.
- This wave does **not** rewrite test contents. It only flips them
  from inert to executing. If a test fails due to runtime
  behaviour, that's a real bug in the migrated code; the worker
  fixes the **migrated code**, not the test (escalation triggers
  below).

### Whitelist + canon

- `grace/frontend.paths` — no edit. `lib/grace/**` and
  `components/grace/**` already cover test files.
- `docs/GRACE_CANON.md` — append a one-paragraph subsection §13.X
  "JS test gate" stating: vitest runs on `lib/grace/**` and
  `components/grace/**`; legacy is excluded; failure halts
  bootstrap. Additive only.

### Negative-test extension

- `scripts/grace/check-negative.sh` — extend with two new probes:
  - NEG-VITEST-1: insert `expect(1).toBe(2)` into a temp test;
    Gate 11 must fail and name the file. Restore; gate green.
  - NEG-VITEST-2: insert a syntax error into the same file;
    Gate 11 must fail with a parse error. Restore; gate green.
- This script is already an append-only artifact owned by W-2.0;
  this wave appends two probes, never edits existing ones.

## Frozen / Out Of Scope

- `legacy/frontend/**` — read-only. `legacy/frontend/vitest.config.ts`
  is referenced for shape only; no rule says the new config must
  match it.
- `apps/api/**` — backend. pytest gate is unchanged.
- `packages/contracts/**` — frozen.
- Existing gates 1–10 in `bootstrap.sh` — frozen.
- Playwright / e2e tests — out of scope. A separate wave introduces
  them. The `playwright.config.ts` present in `legacy/frontend/` is
  not migrated here.
- Coverage thresholds, mutation testing, snapshot policies — out
  of scope. The gate ships at "all green or red", no thresholds.
- New product tests beyond the ported reducer/pure-function tests.
  Adding a new test file in this wave is forbidden; this wave is
  about the runner.

## Must Preserve

- INV-MARKER-COMPLETE: every test file under whitelist already
  carries its markers (W-2.4 / W-2.7 produced them). Gate 9 stays
  at 0.
- INV-CONTRACTS-ONLY-IMPORT: tests import payload types from the
  W-2.1 contract source only. Gate 8 stays at 0.
- INV-DRIFT-CLEAN.
- INV-LEGACY-INERT: vitest config explicitly excludes
  `legacy/**`. Tests inside legacy do not run.
- INV-GATE-APPEND-ONLY: `bootstrap.sh` block of gates 1–10 is not
  reordered or weakened. Only Gate 11 is appended.
- INV-NO-NEW-TESTS: the wave does not add new product tests; only
  config + gate + canon paragraph + negative-test probes.
- INV-IDEMPOTENT-GATE: rerunning Gate 11 on a clean tree exits 0
  twice in a row.

  fixture into the test file's sibling `__fixtures__/` directory
  inside the GRACE perimeter. Do not import from `legacy/**`.
