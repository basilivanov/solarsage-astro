## Goal

Enable browser state debugging and headless testing for frontend packet execution.

Currently, when packets modify frontend code, there is no way to verify browser
state, test UI rendering, or debug frontend behavior in headless mode. This wave
establishes:

1. **Playwright E2E infrastructure** with mock backend
2. **Correlation ID round-trip** (browser → backend → logs)
3. **E2E packet runner** integration with evidence collection
4. **Test instrumentation gates** for frontend verification

After this wave, packets can prove frontend changes work through automated headless
tests, not just "it compiles" checks.

- INV-CORRELATION-UUID: Correlation IDs are UUID v4, injected on every fetch as `X-Correlation-Id` header.

- INV-BOUNDED-TESTS: Tests are bounded (max 5 minutes per spec, max 30 seconds per test).
- INV-NO-EXTERNAL-DEPS: No external dependencies (no Sentry, no third-party log shipping).

## Required Design Decisions

### Decision 1: Playwright Test Projects

**Options:**
- **Option A**: Single project with all tests
- **Option B**: Three projects (fixtures-only, mock-backend, error-states) ← RECOMMENDED
- **Option C**: Two projects (fixtures-only, integration)

**Recommendation**: **Option B** (following W-1.5 pattern)

**Rationale**:
- `fixtures-only`: Visual snapshot parity with mock data
- `mock-backend`: Real fetch calls against in-process mock server
- `error-states`: 401/422/500 error UI rendering
- Clear separation of concerns, easier to debug failures

### Decision 2: Mock Backend Implementation

**Options:**
- **Option A**: Express in-process server ← RECOMMENDED
- **Option B**: MSW (Mock Service Worker)
- **Option C**: Playwright route interception

**Recommendation**: **Option A** (Express in-process)

**Rationale**:
- Reuses existing JSON fixtures from `tests/fixtures/*.json`
- Supports correlation ID echo (returns `X-Correlation-Id` in response)
- Supports error injection (401, 422, 500 via query params)
- Easier to debug than MSW or route interception

### Decision 3: Correlation ID Injection Point

**Options:**
- **Option A**: Fetch wrapper in `lib/api/_client.ts` ← RECOMMENDED
- **Option B**: Global fetch interceptor
- **Option C**: Per-endpoint manual injection

**Recommendation**: **Option A** (Fetch wrapper)

**Rationale**:
- Centralized injection point
- Already exists in `lib/api/_client.ts`
- No global state pollution
- Easy to test

### Decision 4: E2E Packet Runner Integration

**Options:**
- **Option A**: Standalone script `scripts/e2e_packet_runner.py` ← RECOMMENDED
- **Option B**: Integrated into `scripts/guardrails.sh`
- **Option C**: New npm script only

**Recommendation**: **Option A** (Standalone script)

**Rationale**:
- Can be called from GRACE orchestrator
- Wraps Next.js dev server + mock backend + Playwright
- Collects evidence artifacts (playwright-report/, test-results/)
- Clean shutdown on exit

### 4. Update Correlation Middleware

```typescript
import { v4 as uuidv4 } from 'uuid';

export function generateCorrelationId(): string {
  return uuidv4();
}

export function injectCorrelationId(headers: Headers): void {
  if (!headers.has('X-Correlation-Id')) {
    headers.set('X-Correlation-Id', generateCorrelationId());
  }
}

export function extractCorrelationId(response: Response): string | null {
  return response.headers.get('X-Correlation-Id');
}
```

### 5. Update API Client

Update `lib/api/_client.ts`:

```typescript
import { injectCorrelationId } from '../log/correlation';
import { log } from '../log';

export async function fetchAPI<T>(endpoint: string, options?: RequestInit): Promise<T> {
  const headers = new Headers(options?.headers);
  injectCorrelationId(headers);

  const correlationId = headers.get('X-Correlation-Id')!;

  log({
    module: 'M-FRONTEND-API-CLIENT',
    fn: 'fetchAPI',
    block: 'HTTP_REQUEST',
    event: 'fetch_started',
    result: 'ok',
    correlation_id: correlationId,
    timestamp: new Date().toISOString(),
    level: 'info',
    payload: { endpoint },
  });

  try {
    const response = await fetch(`${process.env.NEXT_PUBLIC_API_URL}${endpoint}`, {
      ...options,
      headers,
    });

    if (!response.ok) {
      log({
        module: 'M-FRONTEND-API-CLIENT',
        fn: 'fetchAPI',
        block: 'HTTP_REQUEST',
        event: 'fetch_failed',
        result: 'fail',
        correlation_id: correlationId,
        timestamp: new Date().toISOString(),
        level: 'error',
        payload: { endpoint, status: response.status },
      });
      throw new Error(`HTTP ${response.status}`);
    }

    const data = await response.json();

    log({
      module: 'M-FRONTEND-API-CLIENT',
      fn: 'fetchAPI',
      block: 'HTTP_REQUEST',
      event: 'fetch_completed',
      result: 'ok',
      correlation_id: correlationId,
      timestamp: new Date().toISOString(),
      level: 'info',
      payload: { endpoint },
    });

    return data;
  } catch (error) {
    log({
      module: 'M-FRONTEND-API-CLIENT',
      fn: 'fetchAPI',
      block: 'HTTP_REQUEST',
      event: 'fetch_failed',
      result: 'fail',
      correlation_id: correlationId,
      timestamp: new Date().toISOString(),
      level: 'error',
      payload: { endpoint, error: String(error) },
    });
    throw error;
  }
}
```

### 6. Create Backend Correlation Middleware

Create `apps/api/app/middleware/correlation.py`:

```python
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request
import logging

logger = logging.getLogger(__name__)

class CorrelationMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        correlation_id = request.headers.get("X-Correlation-Id")
        
        # Inject into logger context
        if correlation_id:
            logger.info(
                "Request received",
                extra={
                    "correlation_id": correlation_id,
                    "path": request.url.path,
                    "method": request.method,
                },
            )
        
        response = await call_next(request)
        
        # Echo in response
        if correlation_id:
            response.headers["X-Correlation-Id"] = correlation_id
        
        return response
```

Register in `apps/api/app/main.py`:

```python
from app.middleware.correlation import CorrelationMiddleware

app.add_middleware(CorrelationMiddleware)
```

```typescript
import { test, expect } from '@playwright/test';

test.describe('Correlation ID round-trip', () => {
  test('frontend sends correlation ID, backend echoes it', async ({ page }) => {
    // Intercept API calls
    const requests: string[] = [];
    const responses: string[] = [];

    page.on('request', (request) => {
      const correlationId = request.headers()['x-correlation-id'];
      if (correlationId) {
        requests.push(correlationId);
      }
    });

    page.on('response', (response) => {
      const correlationId = response.headers()['x-correlation-id'];
      if (correlationId) {
        responses.push(correlationId);
      }
    });

    await page.goto('/today');
    await page.waitForLoadState('networkidle');

    // Check correlation IDs match
    expect(requests.length).toBeGreaterThan(0);
    expect(responses.length).toBeGreaterThan(0);
    expect(requests[0]).toBe(responses[0]);
  });
});
```

### 8. Create E2E Packet Runner

Create `scripts/e2e_packet_runner.py`:

```python
#!/usr/bin/env python3
"""E2E packet runner with Playwright integration."""

import subprocess
import sys
import time
from pathlib import Path

def start_next_dev():
    """Start Next.js dev server."""
    proc = subprocess.Popen(
        ["pnpm", "dev"],
        stdout=subprocess.PIPE,
        stderr=subprocess.PIPE,
    )
    time.sleep(5)  # Wait for server to start
    return proc

def start_mock_backend():
    """Start mock backend server."""
    proc = subprocess.Popen(
        ["node", "tests/fixtures/server.ts"],
        stdout=subprocess.PIPE,
        stderr=subprocess.PIPE,
    )
    time.sleep(2)  # Wait for server to start
    return proc

def run_playwright_tests():
    """Run Playwright tests."""
    result = subprocess.run(
        ["npx", "playwright", "test", "--project=mock-backend"],
        capture_output=True,
        text=True,
    )
    return result.returncode == 0

def main():
    print("Starting Next.js dev server...")
    next_proc = start_next_dev()
    
    print("Starting mock backend server...")
    mock_proc = start_mock_backend()
    
    try:
        print("Running Playwright tests...")
        success = run_playwright_tests()
        
        if success:
            print("✓ All tests passed")
            return 0
        else:
            print("✗ Tests failed")
            return 1
    finally:
        print("Shutting down servers...")
        next_proc.terminate()
        mock_proc.terminate()
        next_proc.wait()
        mock_proc.wait()

if __name__ == "__main__":
    sys.exit(main())
```
