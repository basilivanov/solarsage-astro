# W-5.2 — today_payloads cache + meta.cached

status: completed
phase: PHASE-5-LLM
wave: W-5.2
modules: M-TODAY-SERVICE

---

## Decision

Cache LLM-generated `TodayPayload` responses to avoid redundant LLM calls. Add `meta.cached` flag to payload to indicate cache hit. Cache keyed by (user, date).

**Why:** LLM calls are expensive (latency + cost). Caching enables instant responses for repeated requests and calendar views. `meta.cached` flag enables frontend to show cache status.

---

## Acceptance Criteria

- [x] `TodayService` caches LLM payloads
- [x] Cache keyed by (user_id, date)
- [x] `meta.cached` flag added to `TodayPayload` schema
- [x] Cache invalidation on profile updates
