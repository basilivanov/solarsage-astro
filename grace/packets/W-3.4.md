# W-3.4 — solarsage_client + natal/period/daily snapshot caches

status: completed
phase: PHASE-3-SOLARSAGE
wave: W-3.4
modules: M-SOLARSAGE-CLIENT, M-CACHE

---

## Decision

Create `solarsage_client.py` in main API to communicate with sidecar service. Implement caching layer for natal charts, period transits, and daily snapshots to reduce redundant calculations.

**Why:** Main API needs a clean interface to the sidecar. Caching prevents recalculating expensive astrological data for repeated requests.

---

## Acceptance Criteria

- [x] `solarsage_client.py` wraps HTTP calls to sidecar
- [x] Client handles connection errors gracefully
- [x] Cache layer stores natal charts by user profile
- [x] Cache layer stores daily snapshots by (user, date)
- [x] Cache invalidation strategy defined
