# W-4.0 — Scoring canon data schemas and v1 YAML

status: completed
phase: PHASE-4-SCORING
wave: W-4.0
modules: M-SCORING-CANON

---

## Decision

Define scoring canon data structures as Pydantic schemas. Establish v1 scoring rules in YAML format (or embedded in Python) that map astrological signals to sphere scores and day status calculations.

**Why:** Scoring logic must be data-driven and versioned. Separating rules from code enables non-engineers to review and update scoring weights without touching implementation.

---

## Acceptance Criteria

- [x] Scoring canon schemas defined (signal types, sphere mappings, weights)
- [x] v1 scoring rules established (aspect → sphere mappings)
- [x] Rules cover all major aspects (trine, sextile, square, opposition, conjunction)
- [x] Rules define sphere categories (career, relationships, health, creativity)

---

## Evidence

- File: `apps/api/app/services/scoring_service.py` — scoring constants (POSITIVE_ASPECTS, NEGATIVE_ASPECTS, NEUTRAL_ASPECTS)
- Implementation: Scoring rules embedded in service logic (W-4.0 establishes structure, W-4.2 implements activation)
- Schema: Sphere categories defined in `_calculate_sphere_scores` method

---

## Negative Tests

- [ ] Invalid aspect type must not contribute to scoring
- [ ] Missing sphere mapping must default to neutral score
- [ ] Scoring rules must be deterministic (same input → same output)
