# W-3.3 — /v1/transits and /v1/range

status: completed
phase: PHASE-3-SOLARSAGE
wave: W-3.3
modules: M-SOLARSAGE-SIDECAR

---

## Decision

Add `/v1/transits` endpoint for single-day transit calculations and `/v1/range` endpoint for multi-day transit ranges. Both endpoints use natal chart data and calculate planetary positions for specified dates.

**Why:** Transits are time-sensitive calculations needed for daily readings. Range endpoint enables efficient batch calculations for calendar views.

---

## Acceptance Criteria

- [x] `/v1/transits` endpoint accepts natal data + target date
- [x] `/v1/range` endpoint accepts natal data + date range
- [x] Both endpoints return transit data in consistent format
- [x] Tests verify transit calculations

---

## Evidence

- File: `apps/solarsage/solarsage/api/transits.py` — transits endpoints
- File: `apps/solarsage/tests/test_transits.py` — transit endpoint tests
- Test: Transit calculations return expected planetary positions
- Test: Range endpoint handles multi-day requests efficiently

---

## Negative Tests

- [ ] Invalid date range (end before start) must return 400 error
- [ ] Range exceeding max days (e.g., >90) must return 400 error
- [ ] Transit calculation with invalid natal data must fail gracefully
