## Goal

`uvicorn app.main:app` starts. `GET /api/health` returns 200 with `version` and
`git_sha`. `alembic upgrade head` succeeds against an empty database. Nothing
else exists yet.

This wave **does not** add any feature endpoint, model, schema, or frontend
change. The frontend keeps running on `NEXT_PUBLIC_USE_FIXTURES=true` and is
not touched at all.

  → if these files exist from earlier exploratory work, they MUST be deleted
    in this wave (see `Drift cleanup` below). They will be re-introduced in
    W-1.2 / W-1.3 / W-1.4 each inside their own packet.

  → same rule: delete any pre-existing file in those directories.

## Must Preserve

- INV-CONTRACT-STABLE: no contract files are touched.
- Boundary "Backend → DB" stays manual SQL/SQLAlchemy Core or SQLModel; no
  ORM auto-REST layer.
- API listens on `:8000`. Sidecar port `127.0.0.1:18091` is mentioned in
  config but **not started** in this wave.
- `.env.example` documents every env var that `core/config.py` reads.

## Drift cleanup (mandatory, part of this wave)

If `apps/api/app/main.py` currently imports any of
`day, calendar, profile, readings, auth` routers, those imports and the
corresponding files MUST be removed in this wave's PR. This is a deliberate
rollback of premature scaffolding so subsequent waves can re-introduce each
file inside its own packet with proper verification.

---

## Verification (every item must pass)

1. `cd apps/api && ruff check .` → exit 0
2. `cd apps/api && mypy app` → exit 0
3. `cd apps/api && alembic upgrade head` against a fresh sqlite DB → exit 0
4. `cd apps/api && uvicorn app.main:app --port 8000 &` then
   `curl -s localhost:8000/api/health` returns
   `{"status":"ok","version":"...","git_sha":"..."}` (200, JSON shape exact).
5. `cd apps/api && pytest tests/test_health.py -q` → all pass.
6. `git grep -nE "from app\.api\.(day|calendar|profile|readings|auth)" apps/api`
   returns nothing.

## Expected Evidence (attach under `## Evidence` below)

- `git diff --stat` of the PR (only files inside `Allowed Write Scope`).
- Output of all 6 verification steps, copy-pasted.
- Confirmation that no file outside `Allowed Write Scope` was modified
  (`git diff --name-only origin/main... | sort` listed in full).

## Escalation triggers (STOP and ask controller)

- You feel the need to add a model, a schema, or any feature endpoint.
- You discover that a file in `Frozen / Out Of Scope` *must* change for the
  build to pass. Do not change it; report it.
- A dependency you want to add is not already implied by `pyproject.toml`'s
  intended runtime (FastAPI, uvicorn, pydantic, pydantic-settings, SQLAlchemy,
  alembic, pytest, ruff, mypy). Anything beyond that list is escalation.
