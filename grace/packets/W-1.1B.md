# Controller Packet — W-1.1B: Contract boundary (source of truth)

## Why this wave exists

The repository currently carries two parallel contract surfaces:

- `lib/contracts/*.ts` — zod schemas (runtime validation + inferred TS types),
  consumed by frontend.
- `packages/contracts/*.ts` — flat TS types, intended as the cross-runtime
  contract but never reachable from Python.

There is no rule for which side is the source of truth, so as soon as W-1.2
adds Pydantic schemas for auth/profile, the three surfaces (Pydantic, zod,
flat TS) will diverge silently. This wave fixes the boundary **before** any
feature code is written. No feature behaviour is added in this wave.

## Decision required from controller (record here, then proceed)

Pick exactly one option and write it under `## Decision` below.

- **A. zod is source of truth.** `lib/contracts/*.ts` exports zod schemas;
  `packages/contracts/schemas/*.json` is generated from them via
  `zod-to-json-schema` and committed; Pydantic models in
  `apps/api/app/schemas/*` are validated against the JSON Schema in CI.

- **B. Pydantic is source of truth.** Pydantic models in
  `apps/api/app/schemas/*` produce `openapi.json` via FastAPI;
  `packages/contracts/*.ts` is generated from it via `openapi-typescript`
  and committed; `lib/contracts/*.ts` is reduced to **form-input zod
  schemas only** (registration, profile edit) and is no longer used to
  validate server responses.

- **C. Dual-master with parity test.** Both sides are hand-written. CI runs
  `tests/contract_parity_test.py` that loads each fixture, asserts FastAPI
  response validates against zod and that JSON Schema generated from each
  side is equal. Manual sync; cheap to start, expensive to maintain.

The remainder of this packet is conditional on the decision. The Allowed
Write Scope below covers the union; worker uses only the subset matching
the chosen option.

## Decision

**Option B — Pydantic is source of truth.**

Обоснование. (1) Backend по канону §6 владеет shape ответа: shape
определяется тем, что сервер действительно возвращает, а не тем, что
фронт согласен принять; делать backend подчинённой стороной (Option A)
противоречит этой собственности. (2) FastAPI и так вынужден держать
Pydantic-модели для сериализации/валидации ответов и для авто-OpenAPI;
держать рядом ещё и zod в качестве "истины" — это вторая копия с
ручной синхронизацией. (3) Option C (dual-master) явно описан в
пакете как "expensive to maintain" и при первой же расходимости
блокирует не один PR, а оба контура; для wave-1, где количество
эндпоинтов будет расти быстро, это неприемлемо. (4) zod на фронте всё
ещё нужен — но только для пользовательского ввода (формы), что
отдельный класс схем, не пересекающийся с ответами API; именно эту
роль за ним и оставляем (`lib/contracts/forms/*.ts`).

Единственный pipeline:
`apps/api/app/schemas/*.py` (Pydantic, `extra="forbid"`, camelCase via
`alias_generator=to_camel`) → `scripts/contracts/export_openapi.py` →
`packages/contracts/openapi.json` → `npx openapi-typescript` →
`packages/contracts/_generated.ts` (banner-stamped) → per-feature
re-export shims `today.ts` / `calendar.ts` / `natal.ts` / `access.ts`.
Drift запрещён CI-гейтом `contracts:check`.

## Evidence

### 1. `pnpm contracts:check` — drift gate

Скрипт регенерирует `openapi.json` и `_generated.ts`, затем требует
чистого `git diff` по этим двум артефактам.

```
$ pnpm contracts:generate
wrote packages/contracts/openapi.json (29832 bytes)
contracts: regenerated openapi.json + _generated.ts

$ git add -A && git commit -qm "regen baseline"

$ pnpm contracts:generate    # повторный запуск
wrote packages/contracts/openapi.json (29832 bytes)
contracts: regenerated openapi.json + _generated.ts

$ git diff --stat
(empty)

$ git diff --exit-code -- packages/contracts/openapi.json packages/contracts/_generated.ts
$ echo "exit=$?"
exit=0
```

Идемпотентность подтверждена: после baseline-регена повторный запуск
даёт байт-в-байт тот же результат, drift-gate выходит с exit 0.

Negative test (вводим искусственный drift):

```
$ python -c "import pathlib,json; p=pathlib.Path('packages/contracts/openapi.json'); d=json.loads(p.read_text()); d['info']['title']='DRIFT'; p.write_text(json.dumps(d, indent=2, ensure_ascii=False, sort_keys=True))"
$ git diff --exit-code -- packages/contracts/openapi.json
- "title": "Astro API",
+ "title": "DRIFT",
$ echo "exit=$?"
exit=1
$ git checkout -- packages/contracts/openapi.json
```

CI воспроизводит то же самое: regen + `git diff --exit-code`.

и **только** туда.
